🗄️ BlueStore — How an OSD Stores Bytes

The complete write path inside a single OSD: ObjectStore Transaction → BlueStore (raw data + RocksDB metadata + WAL) → ACK to primary.

⚙️ Deployment Layout Where each BlueStore component physically lives
💾
Single Device
All BlueStore components on one drive. Simple but bottlenecked by primary device.
💾 💿 ⚡
Tiered (HDD + SSD + NVMe)
Data on HDD, RocksDB on SSD, WAL on NVMe. Production HDD pattern.
